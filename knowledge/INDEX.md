# Knowledge Index

Domain-organized learnings that persist across sessions. See `CLAUDE.md` for the rules/hypotheses/knowledge pattern.

## Domains

- **[mcp-integrations/](mcp-integrations/)** — What works, what breaks, workarounds for MCP servers (Monarch, Schwab, etc.)
- **prompt-design/** — (not yet started) What makes the advisory prompt effective
- **financial-data/** — (not yet started) Patterns in data quality, refresh reliability, etc.

## How to use

1. Before starting a task in a domain, open its `rules.md` (apply by default) and `hypotheses.md` (check if today's work can test one).
2. At end of task, extract insights. Add confirmed facts to `knowledge.md`, unconfirmed to `hypotheses.md`, apply-by-default guidance to `rules.md`.
3. Promote hypothesis → rule after 3+ confirmations. Demote rule → hypothesis on contradiction.
